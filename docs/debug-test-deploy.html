Local-first VS Code assistant

Debug, Test, and Deploy AI Work Assistant

A practical local runbook for bringing up the FastAPI backend, launching the VS Code extension, validating integrations, packaging the extension, and installing it locally.

What Runs Locally

  • VS Code extension in Extension Development Host.
  • FastAPI backend on 127.0.0.1:8765.
  • SQLite database in a local ignored path.
  • Secrets in VS Code SecretStorage, not SQLite.

Required Tools

  • node and corepack
  • pnpm through Corepack
  • uv or python3 -m uv
  • VS Code

Main Checks

  • ./scripts/test.sh
  • ./scripts/lint.sh
  • corepack pnpm typecheck
  • corepack pnpm package:extension

1. Bootstrap the Repo

Run this once after cloning or after dependency changes.

./scripts/bootstrap.sh

If uv is not on your PATH but was installed with pip, the scripts fall back to python3 -m uv.

2. Start the Backend

From the repo root:

./scripts/dev-agent.sh

Expected server:

http://127.0.0.1:8765

Health Check

curl http://127.0.0.1:8765/health

Expected response:

{"status":"ok","version":"0.1.0"}

3. Launch the VS Code Extension

  1. Open ai-work-assistant.code-workspace in VS Code.
  2. Open the Run and Debug panel.
  3. Select Run VS Code Extension.
  4. Press F5.
  5. In the Extension Development Host, open the AI Work Assistant activity bar icon.

The F5 launch config runs the extension build task first. Keep the backend running in a separate terminal if you want sidebar API calls to succeed.

4. Test the Backend API Yourself

Create a Todo

curl -X POST http://127.0.0.1:8765/todos \
  -H "content-type: application/json" \
  -d '{
    "title": "Test local assistant",
    "description": "Created from curl",
    "priority": "medium",
    "source": "manual"
  }'

List Todos

curl http://127.0.0.1:8765/todos

Repo Intelligence

curl http://127.0.0.1:8765/repo/context
curl http://127.0.0.1:8765/repo/architecture
curl http://127.0.0.1:8765/repo/deployment-flow
curl http://127.0.0.1:8765/planner/plan

5. Approval-Gated Actions

Commands and file writes require X-Approval-Decision: approved. Without it, the backend returns an approval-required response and does nothing.

Preview Command Approval Requirement

curl -X POST http://127.0.0.1:8765/repo/commands/run \
  -H "content-type: application/json" \
  -d '{"command": "pytest"}'

Run an Approved Command

curl -X POST http://127.0.0.1:8765/repo/commands/run \
  -H "content-type: application/json" \
  -H "X-Approval-Decision: approved" \
  -d '{"command": "pytest"}'

Supported Repo Commands

  • databricks_bundle_validate
  • pytest
  • ruff
  • mypy

6. Integration Checks

Azure DevOps

Configure aiWorkAssistant.azureDevOps.organization and aiWorkAssistant.azureDevOps.project in VS Code settings. The PAT is requested by the extension and stored in VS Code SecretStorage. 

Command Palette:
Assistant: Sync DevOps Tickets
Assistant: Update Current Ticket

Outlook / Microsoft Graph

The extension asks for a Microsoft Graph access token and stores it in VS Code SecretStorage. Draft responses require explicit approval and are never auto-sent. 

Command Palette:
Assistant: Sync Outlook Emails
Assistant: Create Outlook Draft Response

7. Run the Automated Checks

Command What It Checks
./scripts/test.sh Extension tests and backend pytest suite.
./scripts/lint.sh ESLint, Ruff, and mypy.
corepack pnpm typecheck TypeScript type checking across workspace packages.
corepack pnpm package:extension Builds the VS Code extension VSIX package.

8. Package and Deploy Locally

This project is local-first. Deployment means packaging the VS Code extension and running the backend locally.

Package the Extension

corepack pnpm package:extension

Output:

extension/ai-work-assistant-extension-0.1.0.vsix

Install the VSIX Locally

code --install-extension extension/ai-work-assistant-extension-0.1.0.vsix

Run the Backend Locally

./scripts/dev-agent.sh

Optional Local Config

Create .env or agent/.env from the examples:

cp .env.example .env
cp agent/.env.example agent/.env

9. Troubleshooting

Symptom Check
Sidebar shows backend offline Run ./scripts/dev-agent.sh and verify /health.
F5 does not launch Run corepack pnpm --filter ai-work-assistant-extension build.
Python dependency issues Run uv sync --project agent --extra dev.
VSIX package includes wrong files Check extension/.vscodeignore.
Approval-gated command does nothing Add X-Approval-Decision: approved intentionally.
